# デモ案（問い合わせにつながる設計）

## 1. 泥臭い需要の「洗練された再定義」コピー案

| 現場の悩み（本音） | 洗練された定義（Webコピー） | 戦略的インパクト（融資・プロ向け） |
|---|---|---|
| 日報や報告書を書くのがダルい | 非構造化フィールドデータの即時資産化 | 現場の一次情報を、加工不要な経営データへ変換 |
| あの資料どこだっけ？と探すムダ | 組織知のセマンティック・リトリーバル | 過去10年の全データを「脳の一部」として統合 |
| 同じようなメールを返すのが苦痛 | 文脈適応型ステークホルダー・シンセシス | 相手との関係性と過去ログを汲み取った高度な代筆 |
| 写真撮って、内容をPCで打ち直す手間 | マルチモーダル・データ構造化エンジン | 画像・音声を一瞬で正規のデジタル形式へ同期 |

## 2. デモ制作に直結する「高需要・横展開」シナリオ案

### ① 建設・製造・保守向け：「現場監督の影武者」
- **悩み**: 現場で手が離せず、進捗報告や安全点検の記録が後回しになる
- **デモ内容**: 写真1枚 + 音声メモから、点検報告書と協力会社への指示メールを自動生成
- **刺さる見せ方**: 「スマホだけで完結」「帰社後の事務作業が半減」を先に見せる
- **問い合わせ導線**: 「あなたの帳票フォーマットに合わせた試作を3営業日で提出」

### ② 不動産・士業・商社向け：「10年分の記憶を持つ秘書」
- **悩み**: 過去案件の検索に時間が溶ける
- **デモ内容**: 曖昧な質問（例: 3年前の港区トラブル）から、契約書・メール・チャットを横断検索して要約
- **刺さる見せ方**: 「探す時間0秒体験」を実演し、再現性を示す
- **問い合わせ導線**: 「まずは過去100ファイルでPoC（2週間）」

### ③ 飲食・小売・サービス向け：「クレームをファンに変える返信下書き」
- **悩み**: 忙しい時間帯の長文問い合わせ対応で精神的に消耗する
- **デモ内容**: 問い合わせ文を解析し、怒りのポイントを分解した上で、丁寧かつポリシー準拠の返信案を3秒で提示
- **刺さる見せ方**: 「担当者の負担減」と「炎上予防」を同時に伝える
- **問い合わせ導線**: 「よくある問い合わせ10件で精度検証」

## 3. リテラシーが低い事業者向けのデモ設計原則（最重要）

- **専門用語を禁止**: 「AI」「RAG」よりも「探す時間がなくなる」「文章作成が3秒」を使う
- **先に結果、後で仕組み**: 操作画面より前に「完成物（報告書/返信文/見積書）」を見せる
- **30秒で理解できる構成**: 入力 -> 自動生成 -> 使える成果物の3ステップだけ
- **数字は1つだけ提示**: 「作業時間70%削減」など、覚えやすい数字を1個に絞る
- **操作は2タップ以内**: 写真アップ or 音声入力から開始し、複雑な設定を見せない

## 4. 業種別デモ案（追加）

### 医療・介護
- **テーマ**: 記録業務の時短
- **デモ**: 音声メモから介護記録を自動整形（申し送り文も同時生成）
- **訴求**: 「記録時間を減らし、利用者対応に戻れる」

### 運送・物流
- **テーマ**: 配送トラブル対応の標準化
- **デモ**: 遅延連絡文と社内報告書を同時生成
- **訴求**: 「属人化を防ぎ、誰でも同品質で対応できる」

### 学習塾・スクール
- **テーマ**: 保護者連絡の効率化
- **デモ**: 面談メモから保護者向け報告文を自動生成
- **訴求**: 「誠実な連絡を短時間で、抜け漏れなく」

### 美容・サロン
- **テーマ**: 予約後フォローの自動化
- **デモ**: 施術内容に応じたフォローメッセージを生成
- **訴求**: 「再来店率を上げる接客を仕組み化」

## 5. 機能別デモ案（横展開しやすい）

### A. 音声 -> 文書化
- **例**: 打ち合わせ音声から議事録・ToDo・次回案内を生成
- **対象業種**: 全業種

### B. 画像 -> 報告書化
- **例**: 現場写真から点検報告・修繕提案文を生成
- **対象業種**: 建設、保守、不動産、保険

### C. 問い合わせ文 -> 返信案
- **例**: クレーム/質問に対する複数トーンの返信を生成
- **対象業種**: 小売、飲食、EC、サービス

### D. 社内データ検索 -> 要約
- **例**: 規定・過去資料・メールを横断し、回答案を生成
- **対象業種**: 士業、商社、管理部門

## 6. 「問い合わせしたい」に変える見せ方テンプレート

### デモ冒頭（15秒）
- 「これはAIの説明ではなく、あなたの作業を減らす実演です」

### 実演（45秒）
- 入力は1回（音声/写真/テキスト）
- 出力は2つ（現場でそのまま使える成果物）

### クロージング（15秒）
- 「この形式を、御社の業務に合わせて無料で試作します」
- CTA: 「3営業日で試作デモを受け取る」

## 8. 次に作るべきデモ優先順位（提案）

1. **建設・製造向けの写真/音声 -> 報告書化**（わかりやすくインパクト大）

3. **社内文書検索 + 要約**（経営層に刺さる）
