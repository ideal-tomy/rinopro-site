## 3. AXEONという社名の意味（About ストーリー素材）

社名は2つの読み方を持つ。

- **英語読み「Axle-On（アクスル・オン）」**：軸を作り、駆動させる。物事の中心を見つけて前に動かす。
- **日本語読み「軸（じく）× 苑（えん）」**：軸となる人や物事が集まり、混沌とした事業環境を整える。

代表が大手企業の経営支援や戦略設計の現場で、構想・戦略・要件・実装・運用といった「軸」が分断されている状況を見続けてきた経験から、これらを一つに繋ぐ意志を込めて命名。

> **闇の力も、もう一度。**
> この言葉を、私たちは理念ではなく、実装で証明する。

## 6. 3つの考え方（OUR PRINCIPLES）

### 01. 「実装できる戦略」しか提案しない
事業の中身、業務、組織の力量にあわせて、実現可能な選択肢を提案する。実現できないキレイな絵を描くことはしない。

### 02. PoCで終わらせない、定着まで伴走する
DXが失敗するのは、PoCで満足してしまう構造にある。AXEONは本実装・運用・改善まで併走する。

### 03. 課題の言語化を、最重要視する
経営層と現場、人とAI、技術と業務。それぞれの言葉を一つに繋ぐ精度こそが、プロジェクト成否を分ける。AXEONは「課題の言語化」の品質をプロダクトの品質と同等に扱う。

## 9. 標準体制（OUR TEAM）

**1プロジェクトに対する、標準体制**

| 役割 | 担当領域 |
|---|---|
| 代表 | 戦略設計から実装まで、案件全体に伴走 / 戦略設計と方針策定 / 経営要件の調整 / プロジェクトマネジメント / 要件・課題整理の設計 |
| 専門メンバー | 案件の規模・難易度に応じて最適な専門メンバーをアサイン / AI実装・技術選定 / 開発実装の指揮 / 業務設計 / 既存システムとの統合 |

代表を中心に、案件ごとに最適な専門メンバーをアサインする体制。戦略から実装まで、ワンチームで伴走する。大手SIerのような大人数体制ではなく、営業窓口・PM・実装者でメッセージが薄まることなく、意思決定が早い構造。
