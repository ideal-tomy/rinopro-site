# 実メール送信移行計画（公開後1週間以内）

対象: `api/contact` のログ運用を、実メール送信へ段階移行する。  
候補: Resend または SendGrid

## 1. ゴール

- [ ] 問い合わせ送信時に運用担当へメール通知される
- [ ] 送信失敗時に再送・追跡ができる
- [ ] 既存UI/UX（フォーム見た目・導線）を変えない

## 2. スコープ

### 今回含める

- `src/app/api/contact/route.ts` に送信処理を追加
- APIキーを Vercel 環境変数へ設定
- 送信成功/失敗の監視ログ整備

### 今回含めない

- 問い合わせUIの文言/導線改修
- CRM 連携など大規模運用統合

## 3. 実施タスク（推奨順）

1. プロバイダ選定（Resend / SendGrid）
2. 環境変数追加（例: `RESEND_API_KEY`）
3. 最低限の送信ラッパー実装
4. 失敗時フォールバック（ログ出力継続）を維持
5. 送信可否の運用確認（ステージング/本番）
6. ログ運用手順からメール運用手順へ切替

## 4. 受け入れ条件（Done定義）

- [ ] `api/contact` 成功時に通知メールが届く
- [ ] 失敗時は 5xx を返し、原因追跡可能なログが残る
- [ ] 従来のログ運用も暫定バックアップとして残る
- [ ] 公開判定シートの問い合わせ項目を更新済み

## 5. リスクと緩和

- APIキー設定漏れ
  - 緩和: デプロイ前チェックで必須化
- 送信遅延/失敗
  - 緩和: タイムアウト・再送方針・フォールバックログ
- 迷惑メール判定
  - 緩和: ドメイン認証（SPF/DKIM）と送信元整備
