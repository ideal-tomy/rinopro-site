# AXEON サイト リデザイン方針(2026年版)

## 目的
ターゲットを「中小企業の経営者」から「中堅企業以上のDX推進担当者・経営層」に変更し、
プロジェクト単価の高い案件獲得を狙う。

## ターゲットの再定義

### メインターゲット
- 従業員 200〜3000名規模の中堅企業
- DX推進担当者、IT部門責任者、経営企画
- 年代: 35〜55歳

### サブターゲット
- 大企業(3000名以上)の事業部・新規事業責任者
- 中堅企業の経営層

### 補足
- 小規模事業者の相談も受け付けるが、**メインメッセージでは触れない**
- FAQセクションに1行だけ控えめに記載

## トーンの変更

| 項目 | 旧(小規模向け) | 新(中堅以上向け) |
|---|---|---|
| 雰囲気 | フレンドリー | プロフェッショナル |
| 用語 | 「経営者の方へ」 | 「経営層・DX推進担当者の方へ」 |
| CTA | 「無料で経営相談」 | 「初回コンサルティングのご相談」 |
| 訴求 | 「次の一手を支援」 | 「企業変革の伴走パートナー」 |

## ファーストビューの変更

### 旧
- 主見出し: 「『何が課題かわからない』から、動くシステムまで。」
- サブ見出し: 「戦略コンサルタント × AIトップエンジニアが、一気通貫で。」

### 新(A案)
- **主見出し**: 「戦略実装、一気通貫で。」
- **サブ見出し**: 「戦略コンサルティング経験者 × AI開発のトップエンジニア。」（表記ルールは `docs/strategy.md`）
- **リード文**: 「DX戦略の構想から、AI駆動開発による高速実装、内製化の支援まで。企業変革を『絵に描いた餅』で終わらせない、実装力のあるパートナー。」
- **メインCTA**: 「初回コンサルティングのご相談」
- **サブCTA**: 「サービス詳細」

## 「こんな経営者の方へ」セクションの変更

### セクション見出し
- 旧: 「こんな経営者の方へ」
- 新: 「こんな課題に、お応えします」

### 4つのカードの内容

| | 旧(小規模向け) | 新(中堅以上向け) |
|---|---|---|
| カード1 | DXしたいが、何から手をつければいいか分からない | 全社DX推進が、PoCで止まり実装フェーズに進まない |
| カード2 | コンサルに頼んだら、提案書だけで終わった | 外部ベンダーが乱立し、戦略と実装の整合が取れない |
| カード3 | 開発会社に頼んだら、言った通りにしか作ってくれない | 内製化を進めたいが、技術的な道筋が描けない |
| カード4 | AIを活用したいが、社内に知見がない | 生成AI活用の戦略立案と実装の両輪を担えるパートナーがいない |

## 「なぜAXEONか」3カードの変更

### カード1: 戦略コンサルティング経験に基づく戦略策定
- 大手企業の変革プロジェクトで培った思考フレーム
- 経営課題を技術要件に翻訳
- 投資対効果を見据えた優先順位付け

### カード2: AI駆動開発による高速実装
- 従来のSI開発と比較して大幅な期間短縮
- PoC〜本格実装まで一貫した品質
- 提携技術パートナーとの連携で大規模案件にも対応

### カード3: 戦略・実装・定着までの一気通貫
- ベンダー乱立による責任分散を回避
- 内製化・組織開発まで含めた長期伴走
- 経営層と現場の橋渡し

新しい流れに以下を追加:
- **内製化支援・組織開発**(★大企業向けに重要)

## CTAの変更(全箇所)

| 旧 | 新 |
|---|---|
| 無料で経営相談する | 初回コンサルティングのご相談 |
| 経営相談を予約 | 初回相談を予約する |

## 維持する要素

以下は変更しない:
- 既存のダーク+宇宙感のトーン(プロフェッショナル感と相性◎)
- セクション全体の構成・順序
- ナンバリング(01, 02...)、ラベル等の装飾
- カードのレイアウト
- 配色・フォント

## 実装の順序(段階的に)

### ステップ1: ファーストビュー
最優先。これだけで印象が大きく変わる。

### ステップ2: 「こんな課題に、お応えします」セクション
次に効果が大きい。共感ポイントが変わる。

### ステップ3: 「なぜAXEONか」3カード
強みの説明を中堅以上向けに。

### ステップ4: CTA文言の全体置き換え
全ページのCTAを統一。

### ステップ5: FAQ更新、その他細部
仕上げ。

各ステップごとに**1コミット**を作る。途中で問題があれば前のコミットに戻せる状態を保つ。



## 第3次改修: 視認性とスクロール疲れ対策(モダンSaaS型へ)

### 全体方針
StripeやLinearのようなモダンSaaS UIの「軽やかさ」を取り入れつつ、
既存のダーク+宇宙感のトーンを維持する。

### 1. 「こんな課題に、お応えします」のレイアウト変更

**変更前**: 2列x2行のグリッド  
**変更後**: 
- PC(768px以上): 4列x1行(横並び)
- モバイル(768px未満): 1列x4行(縦並び)

各カードのサイズ感:
- PC: 横幅約 1/4 ずつ、高さ均等
- カード内 padding: 28px
- カード内構成: アイコン → タイトル(18px bold) → 本文(15px regular, line-height 1.7)

絵文字アイコンは、shadcn/ui や lucide-react のアウトラインアイコンに変更。
(💭→MessageCircle, ⚠️→AlertTriangle, 🔧→Wrench, ✨→Sparkles)

### 2. カード本文の視認性向上(全セクション共通)

- セクション説明文(リード): 17px / line-height 1.8
- カード内タイトル: 18〜20px / font-weight 600
- カード内本文: **15〜16px** / line-height 1.7〜1.8
- カード内補足: 13〜14px

特に「なぜAXEONか」3カードと「業種別」「業務別」のカードは、
本文の可読性を最優先する。

### 3. ホバーエフェクトの追加(PCのみ)

カード類すべてに以下のホバー効果を追加:
- transform: translateY(-4px)
- box-shadow を強める
- transition: all 0.2s ease

これによりカードに「触れるもの」感を出し、クリック誘導する。

### 4. セクションナビの強化

ファーストビュー直下にあるセクションナビを以下に改善:
- 背景: rgba(0,0,0,0.6) + backdrop-filter: blur(12px)
- スクロール時に画面上部に sticky で固定
- 文字サイズ: 14px / letter-spacing: 0.05em
- アクティブセクション: アクセント色のアンダーライン
- スクロールプログレスバー(画面最上部に細い線で進捗表示)

### 5. 業種別・業務別セクションの圧縮

#### 業種から探す → タブ切替式
6カードを横並びの「タブ」形式に変更。
クリックされたタブの内容のみ詳細表示。
モバイルではタブを横スクロールに。

#### 業務の型から考える → アコーディオン式
6項目を全て閉じた状態で表示。
クリックで開閉、複数開いてもOK。

### 6. セクション間の視覚的区切り

各セクションの背景に微妙な濃淡を加える:
- 奇数セクション: 既存ダーク
- 偶数セクション: わずかに明るいダーク(明度 +5%)

スクロールしても同じ風景が続く現象を防ぐ。

### 実装の順序

優先順位順:
1. **第3次改修-1**: カード本文の視認性向上(全セクション)
2. **第3次改修-2**: 「こんな課題」を4列横並びに
3. **第3次改修-3**: ホバーエフェクト追加
4. **第3次改修-4**: セクションナビ強化
5. **第3次改修-5**: 業種別タブ化、業務別アコーディオン化
6. **第3次改修-6**: セクション間の視覚的区切り

各改修ごとに1コミット。
