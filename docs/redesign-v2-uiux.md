### 1-1. 配色(Color Tokens)

#### 背景色

```css
/* メイン背景 */
--color-bg-base: #FAFAF8;        /* オフホワイト・少し温かみ */
--color-bg-pure: #FFFFFF;        /* 純白・カード背景や強調用 */
--color-bg-warm: #F5EFE6;        /* 暖色系・「恩」を表現する特別なセクション用 */
--color-bg-neutral: #F4F4F2;     /* 中間色・セクション切替の抑揚用 */
```

**使い分け**:
- ベース全体: `--color-bg-base`
- カード・モーダル: `--color-bg-pure`
- About(社名の由来)など、特別なセクション: `--color-bg-warm`
- セクション間の抑揚: `--color-bg-neutral`

#### 文字色

```css
/* 主要文字 */
--color-text-primary: #1A1A1A;     /* メイン見出し・本文 */
--color-text-secondary: #4A4A4A;   /* サブテキスト・補足 */
--color-text-tertiary: #6B6B6B;    /* キャプション・極小文字 */
--color-text-disabled: #9CA3AF;    /* 無効状態 */
--color-text-on-accent: #FFFFFF;   /* アクセント色背景上の文字 */
```

#### アクセント色(メイン)

```css
/* メインアクセント: ディープブルー(信頼・知性・戦略性) */
--color-accent-primary: #1E3A8A;       /* 通常 */
--color-accent-primary-hover: #1E40AF; /* ホバー時 */
--color-accent-primary-light: #DBEAFE; /* 背景・装飾用の薄い版 */
--color-accent-primary-dark: #1E2966;  /* 強調・深い場面 */
```

**使い分け**:
- プライマリボタン背景: `--color-accent-primary`
- リンク文字: `--color-accent-primary`
- ラベル背景(薄): `--color-accent-primary-light`

#### アクセント色(セカンダリ・暖色)

```css
/* セカンダリアクセント: ゴールド系(恩・温度・日本らしさ) */
--color-accent-warm: #C9A961;          /* メイン */
--color-accent-warm-light: #F5EFE0;    /* 背景・装飾 */
--color-accent-warm-dark: #A88B47;     /* 強調 */
```

**使い分け**:
- About セクションの装飾線
- 「恩送り」を表現する箇所
- 控えめなアクセント装飾

⚠️ **注意**: メインアクセントとセカンダリは**同時に強く使わない**。
セカンダリはあくまで**補助役**。

#### 機能色(Status Colors)

```css
--color-success: #059669;   /* 成功・完了 */
--color-warning: #D97706;   /* 警告・注意 */
--color-error: #DC2626;     /* エラー */
--color-info: #2563EB;      /* 情報・お知らせ */
```

#### ボーダー・区切り線

```css
--color-border-light: #E5E5E0;     /* 薄いボーダー(カード境界) */
--color-border-medium: #D1D1CC;    /* 通常ボーダー */
--color-border-strong: #9CA3AF;    /* 強調ボーダー(フォーカス) */
```

#### 使用比率の目安

```
背景・余白(白系): 70%
文字(黒系): 20%
アクセント色: 7%
セカンダリアクセント: 3%
```

### 1-6. アニメーション(Motion)

#### 基本方針

- **控えめに**: BtoB信頼感重視、派手なアニメーションは避ける
- **意味のあるものだけ**: 装飾アニメは排除、UXに貢献するもののみ
- **速い**: 0.2〜0.3秒以内で完結

#### transition

```css
--transition-fast: all 0.15s ease;
--transition-base: all 0.2s ease;
--transition-slow: all 0.3s ease;
```

#### アニメーションの種類

##### 1. ホバー(全インタラクティブ要素)

```css
.interactive {
  transition: var(--transition-base);
}

.interactive:hover {
  /* カードなら浮き上がり */
  transform: translateY(-2px);
}
```

##### 2. スクロールイン(セクション登場時)

```css
/* Intersection Observer + クラス追加で発火 */
.fade-in {
  opacity: 0;
  transform: translateY(20px);
  transition: opacity 0.6s ease, transform 0.6s ease;
}

.fade-in.is-visible {
  opacity: 1;
  transform: translateY(0);
}

/* 推奨ライブラリ: react-intersection-observer */
```

##### 3. ヒーローのアニメーション

```css
/* 入場時 */
.hero-text {
  animation: fadeInUp 0.8s ease forwards;
}

@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(24px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
```

#### やってはいけないアニメーション

- ❌ 過剰なparallaxスクロール
- ❌ 自動再生される派手な動画背景
- ❌ 1秒以上かかる遷移
- ❌ ユーザー操作を妨げるオーバーレイアニメ
- ❌ 不規則な動き(プロらしさが消える)

## 第2章: トップページLP各セクション詳細

このセクションでは、トップページのLP型1枚構成における全11セクションの詳細仕様を定義する。
各セクションについて以下を明確化:
- 目的(なぜこのセクションが存在するか)
- 配置(画面のどこに置くか、上下のセクションとの関係)
- レイアウト(PC/モバイル)
- コンテンツ(コピーと素材)
- 図解・ビジュアル仕様
- インタラクション
- 実装ヒント

### 実装運用注記(2026-05)
- 現行実装では、**Section 2: About** と **Section 3: Mission** を統合し、単一セクションとして表示する。
- 表示番号は `02` を使用し、Mission の本文(社会的意義・AI活用の本質)を About 本文内に編入する。
- 背景は `--color-bg-warm` を採用し、統合セクション内で「由来→思想→使命」の順に読ませる。
- 将来的に分離表示へ戻す場合は、本章の Section 2/3 定義を基準に再分割する。

#### 図解・ビジュアル仕様

**SVG自作の抽象ビジュアル**を背景に配置。

```
[コンセプト]
「軸」と「前進」を表現する幾何学的な抽象表現

[案A: 上昇する線群]
- 細い線(strokeのみ)が下から上に向かって伸びる
- 線は中心から外側に放射状に広がる
- 動き: 微妙に揺れる(2〜3秒で1サイクル)
- 色: --color-accent-primary を15%透明度で

[案B: 同心円の動き]
- 中心に小さな円(これが「軸」)
- 外側に向かって波紋のように円が広がる
- ゆっくり脈打つ動き(4秒サイクル)
- 色: アクセント色とゴールド系の組み合わせ

[案C: シンプルな線形]
- 左から右に水平に伸びる線(これが「軸」)
- そこから上に向かって短い線が複数本伸びる(「成長」)
- 静的だが、線が時々光るアニメーション

[推奨: 案A]
理由: 「上昇」「前進」のメッセージと整合
     SVGアニメーションで実装しやすい
     PC/モバイル両方で映える
```

#### インタラクション

- スクロールで上にフェードアウト or パララックス
- CTAボタンはホバーで微妙に浮き上がる(transform: translateY(-2px))
- スクロールヒント(下矢印)が控えめに点滅
- ページ読み込み時、コピーが上から順にフェードイン(0.3秒間隔)

#### 実装ヒント

- Heroセクションは `min-height: 100vh` だが、コンテンツが多い場合は柔軟に
- SVGは <svg> インラインで埋め込み(画像ファイルにしない)
- アニメーションは CSS の `@keyframes` で実装
- フォント読み込み完了前は FOUT(Flash of Unstyled Text)対策で `font-display: swap`
