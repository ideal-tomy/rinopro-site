## 1. 大方針転換のサマリー

### 旧方針(refactor-strategy.md)
- ダーク+宇宙感トーン
- 課題解決訴求型(プロフェッショナル路線)
- 体験demoが信頼の補強材料
- 顔出しなし方針
- 「戦略コンサル × AIエンジニア」の能力訴求

### 新方針(redesign-v2)
- **ライトモード**(GembaShift風の親しみやすい温度感)
- **想いと共感起点**(代表のストーリーから始まる)
- **業界別プロダクト例**(動くものを実績として見せる)
- **顔出し + 経歴ガッツリ**(女性経営者の希少性を最大の差別化に)
- **LP型1枚ページ + 詳細下層**(GembaShift構成踏襲)

### 変わらないもの
- ターゲット: 中堅企業以上のDX推進担当者・経営層
- 「Big4」表記は使わない方針
- AIコンシェルジュ機能は残す(UIだけ控えめに)
- /aboutの「3つの考え方」素材は活かす
- /solutions/[業界]の詳細ページは残す

## 2. ターゲットとサイトの役割

### ターゲット

## 12. 配色・タイポグラフィ(ライトモード)

### 配色

```
[背景]
- ベース: オフホワイト #FAFAF8 or #F8F7F4
- セクション切替: 純白 #FFFFFF とオフホワイトで抑揚
- たまに暖色系セクション: #F5EFE6 (恩送りの温度感)

[文字色]
- 主見出し: ダークグレー #1A1A1A
- 本文: ミドルグレー #333333
- 補足: ライトグレー #666666

[アクセント]
- メインアクセント: ディープブルー #1E3A8A or 既存の青を踏襲
  (信頼感、知性、戦略性を表現)
- セカンダリアクセント: ゴールド系 #C9A961 or 暖色
  (恩送り、日本らしさを微妙に表現)

[ボタン]
- プライマリ: メインアクセント色背景 + 白文字
- セカンダリ: 透明背景 + ボーダー + メインアクセント色文字
```

### タイポグラフィ

```
[フォント]
- 日本語: Noto Sans JP / Hiragino Sans
  ※GembaShiftと同じトーン
- 英語: Inter or Plus Jakarta Sans

[サイズ階層]
- ヒーロー主見出し: 48〜64px / bold
- セクション主見出し: 32〜40px / bold
- カードタイトル: 20〜24px / semibold
- 本文: 16〜17px / regular / line-height 1.8
- 補足: 13〜14px / regular

[ウェイト]
- 見出し: 700 (bold)
- 強調: 600 (semibold)
- 本文: 400 (regular)
```

### 余白

```
- セクション間: 上下120px(PC) / 80px(モバイル)
- セクション内: 見出し下48px、本文間24〜32px
- カード内padding: 32px
- 中央寄せmax-width: 1200px(コンテンツ部分)
```

## 14. 下層ページの方針

### /about
LP内のCEO MessageとAboutセクションだけでは入りきらない、
**詳細プロフィール・経歴・想いの長文版**として残す。
LPから「詳細プロフィールを見る」でリンク。

### /services/[各サービス]
LPのOur Solutionsカードから飛ぶ詳細ページ。
各サービスを深掘りして説明。

### /solutions/[業界]
LPのIndustry Showcaseから飛ぶ詳細ページ。
各業界の課題と解決例を詳しく説明。
動くものがあるなら、ここで体験させる。

### /contact
問い合わせフォーム。LPの最終CTAから飛ぶ。

### 削除候補
- /demo, /demo/list(業界別プロダクト例セクションに統合)
- /experience の不要なslug(社内ナレッジBOTと飲食店ダッシュボード以外)
- /flow(LPのApproachセクションに統合)
- /consulting(LPのSolutionsカードに統合 or /servicesに統合)
- /estimate-detailed系(必要性を再判断)

## 15. 実装計画

### ブランチ運用

- 作業ブランチ: `refactor/redesign-v2`
- 元ブランチ: main
- リリース判断: 完成後にPR作成、レビュー経てmainへmerge

### Phase 構成

#### Phase G-1: ライトモードへの基盤切替(2〜3時間・最優先)
- グローバルCSS/Tailwind設定をライトモード前提に
- 共通コンポーネント(Header/Footer)の配色更新
- 既存ページが**最低限見られる状態**を維持

#### Phase G-2: トップページLP型再構築(4〜6時間)
- 11セクションのLP型構造を実装
- 各セクションのコピーと配置
- 動くdemoの「業界別プロダクト例」への組み込み

#### Phase G-3: 顔写真・経歴・代表メッセージ実装(2〜3時間)
- CEO Messageセクションの実装
- 顔写真素材の配置
- 経歴ハイライトの実装

#### Phase G-4: 旧コンテンツの整理・削除(2〜3時間)
- 不要なdemo・コンポーネント・ページの削除
- 残すdemoの整理(社内ナレッジBOT、飲食店ダッシュボード)
- /demo, /experience の整理

#### Phase G-5: 下層ページのライトモード適用(3〜5時間)
- /about, /services, /solutions/[業界], /contact, /experience を
  ライトモードに適応
- LPの構成と整合させる

#### Phase G-6: AI機能の控えめ化(1〜2時間)
- AIに相談FABのサイズ・位置調整
- AI機能を控えめに位置づけ

#### Phase G-7: 動作確認・微調整(2〜3時間)
- 全ページの動作確認
- レスポンシブ調整
- アクセシビリティ確認

### コミット粒度
- 各 Phase ごとに 1〜3コミット
- セクション単位 or 機能単位でコミット
- 大きな変更は段階的に

- Header/Footer/トップCTAは「残すURL」だけを正としてリンクする。
- `/flow` `/consulting` は互換導線として短期的に生かすが、主要導線からは外す。
- チャットや補助導線内の旧URL参照は、段階的に統合先へ置換する。
- 各置換時に「トップ→サービス→問い合わせ」の動線で404がないことを確認する。
