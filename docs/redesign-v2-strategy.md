## 1. 大方針転換のサマリー

### 旧方針(refactor-strategy.md)
- ダーク+宇宙感トーン
- 課題解決訴求型(プロフェッショナル路線)
- 体験demoが信頼の補強材料
- 顔出しなし方針
- 「経営課題の整理・ロードマップ設計とAI実装」の能力訴求（旧文案はコンサル×エンジニア併せ持ち型）

### 新方針(redesign-v2)
- **ライトモード**(GembaShift風の親しみやすい温度感)
- **想いと共感起点**(代表のストーリーから始まる)
- **業界別プロダクト例**(動くものを実績として見せる)
- **顔出し + 経歴ガッツリ**(女性経営者の希少性を最大の差別化に)
- **LP型1枚ページ + 詳細下層**(GembaShift構成踏襲)

### 変わらないもの
- ターゲット: 中堅企業以上のDX推進担当者・経営層
- 「Big4」表記は使わない方針
- AIコンシェルジュ機能は残す(UIだけ控えめに)
- /aboutの「3つの考え方」素材は活かす
- /solutions/[業界]の詳細ページは残す

## 14. 下層ページの方針

### /about
LP内のCEO MessageとAboutセクションだけでは入りきらない、
**詳細プロフィール・経歴・想いの長文版**として残す。
LPから「詳細プロフィールを見る」でリンク。

### /services/[各サービス]
LPのOur Solutionsカードから飛ぶ詳細ページ。
各サービスを深掘りして説明。

### /solutions/[業界]
LPのIndustry Showcaseから飛ぶ詳細ページ。
各業界の課題と解決例を詳しく説明。
必要に応じて `/experience`（LIVE EXPERIENCE / DEMO GALLERY）へ誘導する。

### /experience
demo一覧ギャラリーページとして再構築。
LPのIndustry Showcase末尾CTAから流入し、以下へ分岐させる。
- LIVE EXPERIENCE: `/experience/internal-knowledge-share-bot`, `/experience/restaurant-ops-dashboard-demo`
- DEMO GALLERY: 外部デプロイURL（新規タブ）

### /contact
シンプル版フォームで運用（**v3**）。送信本文は **`message` 主軸**。チャット・詳細見積・業種/パターン入口からは **テキスト prefill**（クエリ `?prefill=`／長文時は sessionStorage 経由の `?prefill=session`）で土台を渡す。旧 **構造化 `?handoff=`** は互換として当面残置。LP の最終 CTA・ヘッダー／フッターからは常にこのシンプル版へ直行する。

### /estimate-detailed（任意ルート）
詳細見積フローは **当面残置**するが、ヘッダー・トップ・チャットの主導線からは外し、希望者のみが到達する **任意ルート**とする。金額ページの「問い合わせに進む」は `/contact` への **テキスト prefill** に統一。問い合わせ前の整理ブロックは **任意**（未整理でも問い合わせ可能）。

### 削除候補
- /demo, /demo/list(業界別プロダクト例セクションに統合)
- /case-studies/[slug](/experience に統合)
- /experience の不要なslug(社内ナレッジBOTと飲食店ダッシュボード以外)
- /flow(LPのApproachセクションに統合)
- /consulting(LPのSolutionsカードに統合 or /servicesに統合)
- /estimate-detailed系（**2026-05 更新**: 当面残置。上記「任意ルート」として運用し、主要導線からは外す）

## 15. 実装計画

### ブランチ運用

- 作業ブランチ: `refactor/redesign-v2`
- 元ブランチ: main
- リリース判断: 完成後にPR作成、レビュー経てmainへmerge

### Phase 構成

#### Phase G-1: ライトモードへの基盤切替(2〜3時間・最優先)
- グローバルCSS/Tailwind設定をライトモード前提に
- 共通コンポーネント(Header/Footer)の配色更新
- 既存ページが**最低限見られる状態**を維持

#### Phase G-2: トップページLP型再構築(4〜6時間)
- 11セクションのLP型構造を実装
- 各セクションのコピーと配置
- 動くdemoの「業界別プロダクト例」への組み込み

#### Phase G-3: 顔写真・経歴・代表メッセージ実装(2〜3時間)
- CEO Messageセクションの実装
- 顔写真素材の配置
- 経歴ハイライトの実装

#### Phase G-4: 旧コンテンツの整理・削除(2〜3時間)
- 不要なdemo・コンポーネント・ページの削除
- 残すdemoの整理(社内ナレッジBOT、飲食店ダッシュボード)
- /case-studies 廃止、/experience 集約導線への整理

#### Phase G-5: 下層ページのライトモード適用(3〜5時間)
- /about, /services, /solutions/[業界], /contact, /experience を
  ライトモードに適応
- LPの構成と整合させる

#### Phase G-6: AI機能の控えめ化(1〜2時間)
- AIに相談FABのサイズ・位置調整
- AI機能を控えめに位置づけ

#### Phase G-7: 動作確認・微調整(2〜3時間)
- 全ページの動作確認
- レスポンシブ調整
- アクセシビリティ確認

### コミット粒度
- 各 Phase ごとに 1〜3コミット
- セクション単位 or 機能単位でコミット
- 大きな変更は段階的に
