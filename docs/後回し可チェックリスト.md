# 後回し可チェックリスト

公開安定後に、UI/UXを維持したまま保守性を上げるための中長期項目です。

## 1. リファクタリング（非機能）

- [ ] `ContactForm` の責務分割（状態管理、初期化、送信処理、表示セクション）
- [ ] API ルート共通のエラーハンドリングユーティリティ化
- [ ] 再試行ロジック（`fetch-*with-retry`）の共通化を検討

## 2. セキュリティ深化

- [ ] CSP の `unsafe-inline` / `unsafe-eval` を削減する実装計画を作成
- [ ] シークレット管理の見直し（短命トークン、鍵ローテーション）
- [ ] 依存ライブラリの脆弱性チェック運用を定期化

## 3. テストと品質基盤

- [ ] 主要 API の契約テスト（入力異常・429・5xx系）を追加
- [ ] 問い合わせ導線のE2E（正常系/失敗系）を追加
- [ ] 回帰確認チェックリストを CI に統合可能な形へ整備

## 4. 可観測性と運用改善

- [ ] エラー監視基盤（外部サービス含む）を導入・統合
- [ ] KPI（問い合わせ完了率、途中離脱、再訪率）を定義し継続観測
- [ ] 週次で改善バックログを更新する運用を定着

## 5. ドキュメント整備

- [ ] 公開運用ルール、障害対応、連絡経路を 1 つの運用ドキュメントに統合
- [ ] 既存の計画書との重複を整理し、参照元を一本化
- [ ] 変更履歴（いつ何を変えたか）を残すテンプレートを整備
