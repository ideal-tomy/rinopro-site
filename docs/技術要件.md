# 技術要件

## 1. 技術スタック概要

| カテゴリ | 技術 | 用途 |
|----------|------|------|
| Framework | Next.js (App Router) | SSR・ルーティング・エッジ関数 |
| Styling | Tailwind CSS + shadcn/ui | 一貫したUI・爆速開発 |
| Animation | Framer Motion / Rive | 遷移・インタラクション・AIキャラ |
| 3D/背景 | Three.js (React Three Fiber) | パーティクル・抽象3D（軽量に） |
| AI | Vercel AI SDK / Gemini | LLM・ストリーミング |
| 音声 | Deepgram | リアルタイム音声認識 |
| DB | Supabase (PostgreSQL + pgvector) | データ・ベクトル検索 |
| CMS | Sanity | 体験・ツールdemoコンテンツ管理 |
| Hosting | Vercel | デプロイ・エッジ |

---

## 2. フレームワーク・ライブラリ詳細

### 2.1 Next.js (App Router)

- **バージョン**: 14.x 以上（Canary含む最新安定版を検討）
- **理由**: SEO、エッジ関数による高速API、App Routerのストリーミング
- **公式**: https://nextjs.org/docs

### 2.2 Tailwind CSS

- **バージョン**: 4.x 推奨（v3互換も可）
- **理由**: デザイン一貫性、開発速度
- **公式**: https://tailwindcss.com/docs

### 2.3 shadcn/ui

- **導入**: コンポーネントをプロジェクトにコピーする方式
- **理由**: カスタマイズ性、デザインの一貫性
- **公式**: https://ui.shadcn.com/docs

### 2.4 Framer Motion

- **バージョン**: 11.x
- **用途**: ページ遷移、ボタンバウンド、スタッガーアニメーション
- **公式**: https://www.framer.com/motion/

### 2.5 Rive

- **用途**: 軽量なインタラクティブAIキャラ/アイコン
- **公式**: https://rive.app/docs

### 2.6 React Three Fiber (R3F)

- **バージョン**: 8.x
- **用途**: 背景のパーティクル・抽象3D（LCP悪化に注意、軽量に）
- **公式**: https://docs.pmnd.rs/react-three-fiber

### 2.7 Vercel AI SDK

- **バージョン**: 4.x
- **用途**: LLMストリーミング、チャットUI
- **公式**: https://sdk.vercel.ai/docs

### 2.8 Deepgram

- **用途**: リアルタイム音声認識（低遅延）
- **公式**: https://developers.deepgram.com/

### 2.9 Supabase

- **用途**: PostgreSQL、pgvector（RAG・履歴分析の将来拡張）
- **公式**: https://supabase.com/docs

### 2.10 Sanity

- **用途**: 事例・demoコンテンツの管理（非エンジニアでも更新可能）
- **公式**: https://www.sanity.io/docs

## 5. バージョン方針

- **メジャー**: 安定版を採用。Canaryは検証後に採用
- **マイナー・パッチ**: セキュリティ・バグ修正は随時取り込み
- **ロック**: `package-lock.json` / `pnpm-lock.yaml` をコミットし、再現性を確保

## 6. 環境変数

| 変数名 | 用途 | 必須 |
|--------|------|------|
| `GOOGLE_GENERATIVE_AI_API_KEY` | Gemini（チャット・ai_live demo） | ○ |
| `DEEPGRAM_API_KEY` | 音声認識 | ○（音声ON時） |
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase | ○ |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase | ○ |
| `SANITY_PROJECT_ID` | Sanity | ○ |
| `SANITY_DATASET` | Sanity | ○ |

### 6.1 Google AI Studio（Gemini API キー取得）

1. [Google AI Studio](https://aistudio.google.com/) にログイン
2. 「Get API key」からキーを作成（プロジェクト紐づけは案内に従う）
3. キーをサーバー専用で扱い、リポジトリにコミットしない
4. ローカルは `.env.local`、Vercel は Project Settings → Environment Variables に設定

---

## 7. パフォーマンス考慮

- **Three.js**: 重い3Dは後段ロード。初期は軽量パーティクルのみ
- **Rive**: Lottieより軽量。必要に応じて遅延ロード
- **画像**: Next.js Image を必ず使用
- **フォント**: `next/font` で最適化
