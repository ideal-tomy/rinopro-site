# 改善効果の計測設計提案

## 目的
- 「良くなった気がする」ではなく、改善の有効性を定量で判定する。
- UI文言変更とプロンプト変更の効果を切り分けて判断する。

## 計測の基本設計
- フェーズ1: トップ改善のみを反映し計測
- フェーズ2: プロンプト改善を追加し再計測
- 変更を同時投入しない（因果を追えるようにする）

## 主要KPI（必須3つ）
- チャット開始率
  - 定義: トップ訪問ユーザーのうちチャット開始した割合
- 2往復継続率
  - 定義: 初回返答後にユーザーが再返信した割合
- 行動化率
  - 定義: 会話後に「次の一手が明確」と判定できる割合

## 補助KPI（あると良い）
- 初回入力完了率
- 初回返答までの時間
- 会話離脱率（1往復以内）

## イベント設計（最低限）
- `top_view`
- `cta_click`
- `chat_open`
- `first_message_sent`
- `assistant_first_reply`
- `user_second_message_sent`
- `session_end`

## 判定ルール（例）
- 2週間または一定セッション数で評価
- 主要KPIのうち2つ以上が改善で採用
- 主要KPIの1つでも悪化幅が大きい場合はロールバック検討

## 分析時の注意
- 新規/再訪ユーザーを分けて見る
- 流入元ごとの差を分けて見る
- 曜日や時間帯の偏りを確認する

## 運用チェックリスト
- [ ] イベント命名が統一されている
- [ ] フェーズごとに変更点が記録されている
- [ ] 主要KPIの定義をチームで合意している
- [ ] ダッシュボードで日次確認できる
