# デモ量産用タクソノミー（重複防止・100本到達後の拡張用）

最終更新: 2026-03-22  
マスターとなる量産パターンは [`docs/新規デモ追加手順.md`](新規デモ追加手順.md) の「量産パターン（横展開）」と整合させる。

## 1) 機能パターン × 入力タイプ（設計軸）

新規1本は、できるだけ **未埋まりのセル** を優先する。セル内の数字は占有の目安（厳密カウントではない）。

| 機能パターン | text_only | audio_text | image_text |
|---|---|---|---|
| 音声→文書化 | （主に audio 列） | 18（+鮮度廃棄・搬入タイムラインほか） | — |
| 画像→報告書化 | 補助可能 | — | 13（+仮設安全巡回ほか） |
| 問い合わせ→返信 | 多数 | 1（留守電メモ→返信草案） | 未開拓 |
| 検索→要約 | 1（記憶の秘書）＋手順・チケット系の要約拡張 | 未開拓 | 未開拓 |
| 帳票・見積・契約 | 多数 | 1（イベント打合せ→見積メモ） | 未開拓 |
| 工程・原価・在庫・品質（オペ） | 多数 | 未開拓 | 未開拓 |

**ルール**: 同じセルに「成果物ペアまで同型」のデモを増やさない。業種は `industryTags` で差別化するか、別セル（入力タイプ・副成果物）を変える。

## 2) 業種タグ（industry スキーマとの付き合い方）

- `industry` は `construction` / `legal` / `manufacturing` のみ（スキーマ制約）
- 小売・医療・物流・不動産・EC 等は **`industryTags` で表現**し、既存デモの横展開として位置づける

## 3) 残りスロット目安（約100本まで）

| 項目 | 目安 |
|---|---|
| 現行シード | 90本 |
| 目標本数 | 約100本 |
| **残り約10本** | batch-08 等で穴埋め |

## 4) バックログ・アイデア（重複チェック後に採用）

以下は **未実装の例**。採用前に [`docs/demo-mock-inventory.md`](demo-mock-inventory.md) と slug 検索で必ず照合する。

### 音声→文書化（audio_text の空き）

- 現場朝礼メモ → 日報への転記補助
- 店舗接客メモ → クレーム・要望の社内共有
- ドライバー報告 → 遅延・事故報告書のたたき台

### 画像→報告（image_text の拡張）

- 外観・設備写真 → 不動産物件状況記録
- 棚卸写真 → 在庫差異メモ
- 納品時写真 → 配送完了報告

### 問い合わせ→返信（text の業種展開）

- 医療系（一般論・注意書き付き）問診前FAQ返信
- 自治体・窓口よくある質問の回答骨子
- B2B見積前の仕様確認返信

### 検索→要約・ナレッジ

- マニュアル検索の要約（製造・CS）
- 過去チケット検索の類似事例サマリ（社内IT）

### オペ・現場（text）

- 教育・研修アンケート → 改善サマリ
- 採用スクリーニングメモ → 書類選考コメント
- 店舗巡回チェック → 改善指示と本部報告
