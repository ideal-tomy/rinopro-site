# デプロイ前に必須チェックリスト

最終公開前に「UI/UX・掲載内容を変えずに」実施する確認項目です。

## 1. 送信・問い合わせ運用

- [ ] `api/contact` の本番運用フローを確定（実メール送信 or 監視通知）
- [ ] 問い合わせをログ運用する場合、運用担当・確認頻度・見逃し時の手順を文書化
- [ ] 送信失敗時の一次対応フロー（再送、手動連絡）を決定

## 2. 個人情報・ログ

- [ ] 本番ログに `name` / `email` / 本文相当をそのまま出さない（マスキング方針を確定）
- [ ] エラーログに機微情報が混ざらないことを確認
- [ ] 必要最小限の監査ログ項目（時刻、経路、結果、requestId）を定義

## 3. セキュリティ最低ライン

- [ ] 現行 CSP の許容理由（`unsafe-inline` / `unsafe-eval`）を明文化
- [ ] 公開直後に実施する CSP 強化計画（Report-Only 含む）を合意
- [ ] 環境変数の本番設定漏れがないことを確認（Sanity / Gemini / Deepgram など）

## 4. 公開判定ゲート

- [ ] `npm run build` が成功する
- [ ] 主要導線（トップ、デモ、問い合わせ、チャット）の手動スモークテストを完了
- [ ] ロールバック判断基準（何が起きたら戻すか）を決定

## 5. 体制

- [ ] 公開当日の責任者・連絡手段・待機時間を決定
- [ ] 問い合わせ急増時の一次受け体制を確保
- [ ] 公開後 24 時間の監視項目（500/429、問い合わせ件数、主要導線離脱）を明確化
