# トップページチャット 分岐設計（確定版）

## 目的

- 来訪者が「何が必要か言語化できていない」状態でも、タップだけで概算・方向性に到達できる導線を作る。
- 最初に知りたい情報（特に金額感）へ短いステップで誘導しつつ、自由記述へいつでも移行できるようにする。
- 1ステップ1質問で負荷を下げ、選択行動の連続で潜在ニーズを想起させる。

## 設計方針

- 1段目は固定5項目（大カテゴリ A〜E）。
- **A・B** は見積もり精度のため **3〜4段** まで深掘りし、回答後に **概算テンプレ** と **問い合わせ（自動見積もり）への引き継ぎ** を行う。
- **C・D・E** は 2段目までとし、**固定メッセージ＋ショートカットボタン** の後に **共通フィニッシュ**（仮見積もりCTA）を必ず表示する。
- 各段で **「その他（自由記述）」** を常設し、離脱ではなく入力切り替えとして扱う。

---

## 共通フィニッシュ文言（必須・全トラック）

> ご入力ありがとうございます。いただいた内容をもとに、**初期検討用の「仮要件定義」と「概算見積もり（前提条件つき）」**を約30秒で作成できます。社内共有用のたたき台としてご活用ください。

- 主ボタン: **無料で仮見積もり・要件定義を作成する**（問い合わせページへ遷移し、選択内容を引き継ぐ）
- サブ: **条件を調整する**（分岐を最初からやり直す）
- サブ: **自由記述で相談する**（チャット入力欄への誘導、必要に応じて文脈を送信）

---

## 1段目（固定5項目）

| ID | ラベル（ボタン文言） |
|----|----------------------|
| A | 開発コストを知りたい |
| B | 契約・コンサル費用を知りたい |
| C | 具体的な開発技術を知りたい |
| D | 実際に得られるツール内容を知りたい |
| E | 依頼方法を知りたい |

## C・D・E（2段＋ショートカット）

2段目回答直後に **固定メッセージ** と **ショートカットボタン** を表示し、続けて **共通フィニッシュ** を必ず表示する。

### C. 具体的な開発技術を知りたい

**C-2**（既存の5＋その他）

- 業務効率化（手作業削減）
- 業務自動化（定型処理）
- 社内ナレッジ共有
- ワークフロー構築
- テキスト自動生成
- その他（自由記述）

**固定メッセージ（例）**: Next.js や Supabase、最新の LLM などを活用し、セキュリティと速度を意識した開発を行っている旨を簡潔に。

**ショートカット**

- [開発スタックを見る] → `/services/development`
- [ツールDemoを見る] → `/demo/list`

### D. 実際に得られるツール内容を知りたい

**D-2**（5＋その他）

- 管理画面（ダッシュボード）
- 社内向けチャットUI
- 自動通知・アラート
- データ連携（既存SaaS / API）
- レポート自動作成
- その他（自由記述）

**固定メッセージ（例）**: 管理画面やチャットUIの挙動をデモで確認できる旨。

**ショートカット**

- [ツールDemo一覧を見る] → `/demo/list`

### E. 依頼方法を知りたい

**E-2**（5＋その他）

- まず相談だけしたい
- 要件が曖昧なまま相談したい
- 相見積もり前提で相談したい
- まずデモを見て判断したい
- 具体案件として進めたい
- その他（自由記述）

**固定メッセージ（例）**: まずは短時間の相談で要件整理が可能な旨（無料の有無は別途コピーで調整可）。

**ショートカット**

- [相談・サービスの流れを見る] → `/consulting`
- [開発の流れを見る] → `/flow`

## 問い合わせページへの引き継ぎ

- 共通CTAで遷移する際、**トラックID・各段の選択ラベル・自由記述**をエンコードして渡す。
- 問い合わせフォームの **メッセージ欄** に、人が読みやすい **仮要件メモ** として展開する（自動生成ページは後続実装でも、引き継ぎデータは先に載せられる）。

---

## 実装メモ（コード側）

- トップ（`pathname === "/"`）かつ **チャット履歴が空** のときのみ、本フローを表示。それ以外は従来のプリセット3問を維持。
- 概算テンプレは **ルールベース** で生成し、法務・営業トーン（断定禁止）に合わせる。
