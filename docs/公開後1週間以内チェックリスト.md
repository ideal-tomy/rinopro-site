# 公開後1週間以内チェックリスト

公開直後の安定運用を優先しつつ、低リスクで改善する項目です。

## 1. 監視と初動改善

- [ ] 5xx / 429 の発生傾向を日次で確認
- [ ] 問い合わせ API の成功率と失敗理由を集計
- [ ] 主要ページの導線離脱ポイントを確認し、運用メモへ反映

## 2. レート制限の実運用対応

- [ ] 現行のメモリ内レート制限の限界を整理（多インスタンス時の挙動含む）
- [ ] 共有ストア型（Redis 等）への移行可否を判断
- [ ] 実施する場合は段階導入計画（検証 -> 本番）を作成

## 3. ログ・エラー可観測性

- [ ] API 応答エラーに `errorCode` / `requestId` を付与する方針を確定
- [ ] 運用ログのフォーマットを統一（検索しやすいキー名へ）
- [ ] 問い合わせ失敗・AI失敗時の調査手順を Runbook 化

## 4. セキュリティ強化準備

- [ ] CSP を Report-Only で検証する計画を開始
- [ ] 不要な許可ドメインや過剰な `connect-src` がないか再確認
- [ ] Deepgram キー運用（公開キー依存の有無）を棚卸し

## 5. 既存UI維持の回帰確認

- [ ] チャット開始・見積・問い合わせ連携のシナリオ回帰を再実施
- [ ] 体験デモ（代表ケース）の表示崩れ・操作不能がないか確認
- [ ] 変更を入れる際は「文言・見た目不変」をレビュー項目に固定
