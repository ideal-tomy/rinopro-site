# 非機能要件

## 1. パフォーマンス

### 1.1 Core Web Vitals 目標値

| 指標 | 目標値 | 備考 |
|------|--------|------|
| LCP (Largest Contentful Paint) | < 2.5s | 75パーセンタイル |
| INP (Interaction to Next Paint) | < 200ms | 100ms単位のレスポンスが理想 |
| CLS (Cumulative Layout Shift) | < 0.1 | レイアウトシフトを防止 |

### 1.2 方針

- **ITに疎い社長** がストレスを感じないため、100ms単位のレスポンスを意識
- **Three.js**: LCP/INP悪化の最大要因になりうるため、軽量パーティクルに抑える
- **画像**: Next.js Image、WebP/AVIF、適切なサイズ指定
- **フォント**: `next/font` で最適化、FOUT/FOIT 防止

### 1.3 計測

- Lighthouse CI
- Vercel Analytics（Web Vitals）
- WebPageTest（定期的）

## 2. SEO

### 2.1 メタタグ

- `title`: ページごとに最適化（60文字以内）
- `description`: メタディスクリプション（160文字以内）
- `robots`: 必要に応じて設定

### 2.2 OGP (Open Graph Protocol)

- `og:title`
- `og:description`
- `og:image`
- `og:type`
- `og:url`

### 2.3 構造化データ

- Organization
- WebSite

### 2.4 技術的SEO

- セマンティックHTML（`<main>`, `<article>`, `<nav>` 等）
- 適切な見出し階層（H1〜H6）
- キャノニカルURL

---

## 3. アクセシビリティ

### 3.1 目標

- **WCAG 2.1 AA** 準拠を目標

### 3.2 必須項目

- キーボード操作: 全機能にアクセス可能
- フォーカスリング: 明確な視覚的フィードバック
- コントラスト比: 本文 4.5:1 以上
- 代替テキスト: 画像・アイコンに `alt` / `aria-label`
- アニメーション: `prefers-reduced-motion: reduce` で軽減

### 3.3 チャットボット

- 音声入力: テキスト入力も併用可能
- スクリーンリーダー対応
- エラー状態の明示

### 3.4 検証ツール

- axe DevTools
- Lighthouse（Accessibility セクション）

---

## 4. セキュリティ

### 4.1 基本方針

- API Key は環境変数で管理（リポジトリにコミットしない）
- サーバーサイドでのみ機密情報を扱う

### 4.2 実装要件

| 項目 | 要件 |
|------|------|
| CSP (Content Security Policy) | 適切なディレクティブ設定 |
| Rate Limiting | チャットAPI・問い合わせAPIに適用 |
| 入力検証 | サニタイズ・バリデーション |
| HTTPS | 全通信で強制 |

### 4.3 問い合わせフォーム

- CSRF 対策
- reCAPTCHA / hCaptcha 等の検討（スパム対策）
- 個人情報の取り扱い明示

---

## 5. 監視・ログ

### 5.1 エラー監視

- **Sentry**: フロントエンド・サーバーエラーの追跡
- エラー発生時のアラート設定

### 5.2 アナリティクス

- **Vercel Analytics**: Web Vitals、ページビュー
- 必要に応じて Google Analytics 4（プライバシー配慮）

### 5.3 ログ

- 本番環境のAPIログは最小限（個人情報を含まない）
- デバッグ用ログは本番では無効化

---

## 6. 可用性・運用

### 6.1 稼働目標

- 可用性: 99.9% 以上（Vercel の SLA に依存）
- ダウンタイム: 計画メンテナンス時のみ

### 6.2 バックアップ

- Supabase: 自動バックアップ
- Sanity: コンテンツのバージョン管理

### 6.3 デプロイ

- Vercel: プレビューデプロイ、本番自動デプロイ
- ブランチ戦略: main → production

- Supabase: プランに応じたスケール
