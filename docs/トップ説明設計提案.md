# トップ説明設計提案

## 目的
- 初見ユーザーが3秒以内に「何のサイトか」「何をすればいいか」「何が得られるか」を理解できる状態を作る。
- チャット開始前の離脱を減らし、最初の会話体験まで到達させる。

## いまの課題仮説
- チャット内の体験は設計されているが、トップでの意味付けが弱く、開始判断が先送りされる。
- 「機能は理解できるが、自分が得する実感」が不足している。

## 提案（ファーストビュー最小構成）
- 一言定義: このサイトが何者かを主語付きで明示する。
- 1分後の価値: ユーザーが得る成果を時間付きで示す。
- 最初の行動: 1行入力など、最小操作を具体的に示す。
- 安心文言: うまく説明できなくても進められると明記する。

## 文言サンプル（そのまま試せる版）
- 見出し: `読むより、話せば進む。`
- サブ: `Axeonは、現場の悩みを会話で整理し、次の一手に変えるAIコンシェルジュです。`
- 補助: `1分で「課題の整理」と「明日やること」が見えます。`
- 入力例: `例）日報作成に毎日30分かかる`
- 安心文言: `うまく説明できなくても大丈夫。質問に答えるだけで整理できます。`

## レイアウト指針
- 見出し/サブ/CTAを1画面内に収める。
- **チャット開始前の「入力例・安心・1分の流し」**は、相談カード内の二重表示ではなく、**全ページ共通の右下「AIに相談」FAB 直上**（[`ConciergeFabNudge`](../src/components/chat/ConciergeFabNudge.tsx)、`nudgePageId` が `home` のとき）に集約する。トップのヒーローと3カードは**何者か・価値・行動**の意味付けに専念する。
- 説明文は最長2行まで。長文は下層に分離する。

## 表示の主導権をユーザーに渡す（再訪配慮）
- 説明ブロックや吹き出しは `最小化` `閉じる` を可能にする。
- 再訪ユーザーでは、初期状態を最小化してノイズを減らす。
- 一度閉じた状態は保存し、次回訪問でも尊重する。

## モバイル表示最適化
- 見出しは1行-2行で収め、サブコピーを短文化する。
- 吹き出し文言は1-2行に制限し、長文を避ける。
- CTAの上に情報を積みすぎず、スクロールなしで行動導線を見せる。

## 再訪ユーザー向け頻度制御
- 初回訪問: 説明要素をフル表示する。
- 2-3回目: 要点のみ表示する（短文 + CTA）。
- 4回目以降: 説明をコンパクト表示し、必要時に展開できるようにする。

## 会話以外の探索導線を同時に提示
- チャット開始以外に `自分で見る` 導線を明示する。
- 例: デモ一覧、サービス紹介、問い合わせへのショートカット。
- 「会話したくない」ユーザーにも回遊の逃げ道を残す。

## ページ横断での整合方針
- トップ、demo、demo一覧、サービス、問い合わせでUIの見た目を統一する。
- 文言はページ目的に合わせて差し替える（同一UI、文脈別コピー）。
- トップは意味付け重視、下層は行動補助重視で役割を分ける。

## 表示優先度（実装順）
- 1位: 何者か + 1分後の価値 + 最初の行動
- 2位: 入力例 + 安心文言
- 3位: 表示制御（最小化/閉じる）
- 4位: モバイル短文化
- 5位: 再訪頻度制御
- 6位: 会話以外の探索導線
- 7位: ページ別文言最適化

## A/Bテスト案（拡張）
- A: 現状（固定説明）
- B: 短文化したトップ説明 + 安心文言
- C: B + 表示制御 + 再訪頻度制御
- 比較指標: CTAクリック率、チャット開始率、再訪時離脱率、回遊率

## 実装チェックリスト
- [ ] トップで「何者か」「価値」「最初の行動」の3点が同時に見える
- [ ] 右下FAB 直上の `home` 用吹き出し（または方針どおり）で入力例・安心が伝わる
- [ ] 安心感が伝わる（失敗不安を下げる）
- [ ] ファーストビューで読み切れる文字量に収まる
- [ ] 説明ブロック/吹き出しの最小化または閉じる操作がある
- [ ] モバイルでテキスト量が1-2行中心に最適化されている
- [ ] 再訪時の表示頻度制御が効いている
- [ ] 会話以外の探索導線がある
- [ ] ページ横断でUI統一、文言最適化ができている

## 成功指標（最低限）
- トップ到達からチャット開始までの到達率
- トップ滞在後の離脱率
- CTAクリック率
- 再訪時の離脱率
- チャット非利用ユーザーの回遊率

## Cursor への実装依頼（本ドキュメントの範囲：トップ説明まわり）

### 手順
1. **先に** `docs/トップ説明設計提案.md` と `docs/チャット開始率を上げるUIUX提案.md` の **両方** を前提に、Cursor で **PLAN（実装方針・差分箇所・受け入れ条件）** を書かせる。
2. 内容を人間で確認したあと、**同じ会話 or 専用ブランチ** で実装に進む。

### PLAN 作成依頼文（短く使う版・本ファイル向けコピー用）

次を Cursor（Plan モード推奨）に貼り、**まずは PLAN だけ** を出力させる。

> `docs/トップ説明設計提案.md` を実装方針の土台にし、初見向けの「何者か / 1分後の価値 / 最初の行動 / 安心」をトップのファーストビュー（ヒーロー＋3カード＋`自分で見る`）に反映する。  
> **チャットを開く前の先回り**（吹き出し内のローテ・1分ステップ・閉じる/最小化等）の**実体**は右下グローバル FAB 付近（`docs/チャット開始率を上げるUIUX提案.md` の「実装メモ」）を前提に、トップ専用コピーは `site-copy` の `nudgePageId` `home` で調整する。  
> 手順: **(1) 既存のトップページのコンポーネント構成を調査 (2) コピーとレイアウト差分案 (3) トップ文脈とFAB案内の役割の棲み分け (4) 受け入れ条件と計測** まで含めた **PLAN** を書く。コード変更は **PLAN 承認後**。

**完全版の依頼文**（手順の細部・チェックリスト付き）は `docs/チャット開始率を上げるUIUX提案.md` 末尾の「Cursor への PLAN 作成依頼文（トップ＋チャット開始率UI）」をコピーして使うこと。
