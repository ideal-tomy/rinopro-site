# AXEON コピー・表記ルール（要約）

サイト・metadata・会社概要で共有する表記方針。法務・事実と齟齬が出ないよう、**経歴・チーム定位**と**サービス名**を分けて扱う。

## 1. 避ける表現（経歴・チームの文脈）

次の語は、**特定の組織イメージ（いわゆる戦略コンサルファーム等）を連想させやすい**ため、人物・チーム紹介・沿革・FAQの事実ベースの文では使わない。

- 「戦略コンサルティングファーム」「戦略ファーム」
- 「戦略コンサルティング経験者」（略の「戦略コンサル」含む）
- 業態の名指しで対比する「戦略コンサルでは〜」など

## 2. 推奨表現（能力・実務ベース）

- **タグライン（左側）**: 「経営課題の整理とロードマップ設計に強いコンサルティング経験者 × AI開発のトップエンジニア」
  - 長すぎる箇所では短縮可: 例「経営課題の整理とロードマップに強いコンサル経験者 × AI開発のトップエンジニア」（用途に合わせ調整）
- **培ってきた強み**: 「経営と現場の『翻訳』で培ってきた視点」など、**翻訳・伴走**の語彙で統一
- **会社・チーム一文**: 「経営・事業の視点とAI実装力」「上流設計の視点とAI実装力」など、**ファーム名に依存しない**言い方
- **他社・従来型との対比**: 「提案や上流設計が中心の支援では〜」「別チームに実装が渡る従来型では〜」のように、**業態ラベルに依存しない**書き方

## 3. サービス名（サイト内の正式表記）

- **DX戦略設計** を用いる（`/services/consulting` および `service-offerings` の `dx-strategy` と整合）。
- 「DX戦略コンサルティング」は新規コピーでは使わない（旧ドキュメントは `docs/redesign-v2-*.md` の改訂版に追随）。

## 4. 変更時のチェック

- リポジトリ全体検索: `戦略コンサル` `コンサルティングファーム` `DX戦略コンサルティング`
- 代表・事実に関わる文は、**一文ごとに最終確認**してから公開する。
