# 公開判定シート（Vercel / 現状維持）

本シートは「UI/UX・掲載内容を変更しない」前提で、公開可否を判定するための実行用チェックです。

## 1) 判定ゲート（必須）

### Build

- [x] `npm run build` 実行済み（2026-04-17）
- [x] 終了コード `0`（成功）
- [ ] Build 警告の取り扱いを合意（ロックファイル複数検出の警告）

### 導線スモーク（本番URLで最終確認）

- [ ] `/` が表示される
- [ ] `/demo` と `/demo/[slug]` が表示される
- [ ] `/experience` と `/experience/[slug]` が表示される
- [ ] `/contact` の入力〜送信（ログ運用）まで完了できる
- [ ] チャット導線（`/api/chat`）で応答を受け取れる

### API健全性

- [ ] `/api/contact` が `429` 以外で異常応答を返していない
- [ ] `/api/chat` が連続失敗していない
- [ ] `/api/demo/run` / `/api/estimate-detailed` / `/api/inquiry-brief` が主要シナリオで成功

## 2) 公開停止条件（1つでも該当で停止）

- [ ] ビルド失敗（終了コード非0）
- [ ] 主要導線で進行不能、または表示崩れが再現する
- [ ] 問い合わせ導線が成立しない（送信できない、運用ログで追跡不能）
- [ ] 5xx が連続し、短時間で復旧見込みがない

## 3) ロールバック判断基準

- [ ] 主要導線停止が 15 分以上継続
- [ ] 問い合わせ不能が 10 分以上継続
- [ ] チャットAPI失敗率が急増し、代替導線で吸収不能
- [ ] 担当者判断で「利用者影響が顕在化」と判定

## 4) 連絡体制（公開当日）

- [ ] リリース責任者
- [ ] 一次対応担当（問い合わせ）
- [ ] 一次対応担当（技術）
- [ ] 連絡チャネル（Slack/電話）
- [ ] 待機時間（公開後24時間）
