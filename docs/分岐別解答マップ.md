# AIコンシェルジュ 分岐別解答マップ

## 目的

- どの分岐でどの解答が出るかを可視化する。
- 「寄り添い」と「次の行動」まで一貫した体験にする。
- 更新時に、回答欠落や導線欠落を防ぐ。

> **位置づけ**: 本書は [`要件定義.md`](./要件定義.md) §3 のチャット詳細（分岐・文言・結果 UI 導線）のソースの一つ。トップ分岐の操作設計・CTA は [`top-chat-branching-design.md`](./top-chat-branching-design.md) を先に読む。

---

## ルート分岐（ROOT_CHOICES）

| root optionId | 表示文言 | 次ステップ |
|---|---|---|
| `root_a` | 開発コストの概算を知りたい | `A3` → `A_SCOPE` → `buildEstimateBlockA` |
| `root_b` | コンサル・伴走の概算を知りたい | `B2` → `B_SCOPE` → `buildEstimateBlockB` |
| `root_cde` | 技術・ツール・進め方を知りたい | `CDE_PICK` → `C2 or D2 or E2` → `buildCdeSummaryBlock` |
| `root_e` | まず相談・窓口の進め方を知りたい | `E2` → `buildCdeSummaryBlock("E")` |

## 出力順ルール
- A/B: 概算レンジ → ひとことの整理 → 前提条件 → 次の一歩 → 選択内容（参考） → 免責
- C/D/E: ご案内 → ひとことの整理 → 次の一歩 → 選択内容（参考） → 免責

## UI導線（結果画面）

- 結果本文内の「次の一歩」テキストは、表示時にスクロール領域から分離される。
- 結果カード直下に、2カラムの導線ボタンを表示する。
  - 左: `demoを体験`（分岐に応じて `/experience/{slug}` または `/demo/list`）
  - 右: `要件をまとめてみる`（`/estimate-detailed` へコンテキスト付き遷移）
- 「次の一歩」という見出し文言は、UI上には表示しない。
