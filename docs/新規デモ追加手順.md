# 新規デモ追加手順（Sanityだけで完結）

新規デモは **Sanity Studio でレコードを追加するだけ** で量産できます。コード変更は不要です。

## 前提

- `/studio` で Sanity Studio にアクセスできること
- `aiDemo` 型のドキュメントを作成する

**重複防止**: 新規作成前は必ず [`docs/demo-mock-inventory.md`](demo-mock-inventory.md) で slug・テーマ・成果物ペアの重複を確認し、追加後は同ファイルを更新する。シード用のオブジェクト形式は [`docs/demo-next-20-seed-format.md`](demo-next-20-seed-format.md) を参照。

## aiDemo 入力テンプレート（運用定義）

### 必須項目

| 項目 | 説明 | 例 |
|------|------|-----|
| **title** | デモの表示名 | 現場監督の影武者 |
| **slug** | URL 用（`/demo/[slug]`） | construction-shadow-foreman |
| **industry** | construction / legal / manufacturing | construction |
| **inputType** | text_only / audio_text / image_text | audio_text |
| **runMode** | ai_live（実AI） / mock_preview（モック） | mock_preview |
| **systemPrompt** | LLM に渡す指示（ai_live 時必須） | あなたは建設現場の... |
| **mockOutputPrimary** / **mockOutputSecondary** | モック出力（mock_preview 時必須） | 成果物テキスト |

### 推奨項目（問い合わせ導線・体験価値のため）

| 項目 | 説明 |
|------|------|
| **outputStructure** | 成果物2点を必須化する見出し・フォーマット指示 |
| **writingTone** | 文体プリセット（`default` / `real_estate` / `legal` / `care` / `b2b_ops` / `cs_support`）。モック口調と **ai_live 時の system 追記**に使用。未設定は `default` |
| **sampleData** | ワンクリック投入用。実演用3パターン推奨 |
| **ctaTitle** | 導線エリアの見出し（「3営業日で試作」など） |
| **ctaButtonText** | ボタンラベル（例: 3営業日で試作デモを受け取る） |
| **oneLiner** | 15〜24文字で即理解できる一言説明 |
| **inputPlaceholder** | 入力欄のヒントテキスト |

### 一覧表示用（任意）

| 項目 | 説明 |
|------|------|
| description | デモの説明文 |
| image | サムネイル |
| functionTags / industryTags / moduleTags | 一覧・絞り込み用 |
| storyLead | 詳細表示用 |

## 手順

### 1. Sanity Studio を開く

`https://あなたのサイト/studio` にアクセスし、ログインします。

### 2. AI Demo を新規作成

1. 左サイドバーで **AI Demo** をクリック
2. **Create** または **+** で新規ドキュメントを作成
3. 必須項目から順に入力

### 3. 必須項目（上記テンプレート参照）

| 項目 | 説明 | 例 |
|------|------|-----|
| **タイトル** | デモの表示名 | 現場監督の影武者 |
| **スラッグ** | URL 用（`/demo/[slug]`） | construction-shadow-foreman |
| **業種** | construction / legal / manufacturing | construction |
| **入力タイプ** | text_only / audio_text / image_text | audio_text |
| **システムプロンプト** | LLM に渡す指示 | あなたは建設現場の... |

### 4. 推奨項目

| 項目 | 説明 |
|------|------|
| 出力構造指示 | 成果物2点を固定する見出し・フォーマット |
| サンプルデータ | ワンクリック投入用。3件推奨 |
| CTA見出し | 導線エリアの見出し |
| CTAボタン文言 | ボタンラベル（例: 3営業日で試作デモを受け取る） |

### 5. 一覧表示用（任意）

| 項目 | 説明 |
|------|------|
| 説明 | デモの説明文 |
| 画像 | サムネイル |
| 機能タグ / 業種タグ / モジュールタグ | 一覧・絞り込み用 |
| 一言説明 | カード表示用 |
| ストーリーリード | 詳細表示用 |

### 6. 公開

右上の **Publish** をクリックして公開します。

## 確認

- `/demo` 一覧に表示される
- `/demo/[スラッグ]` で詳細ページが開く
- デモ実行パネルで入力・ストリーミング出力が動作する

## 初回セットアップ（高品質3本を一括投入）

以下のコマンドで高品質3デモを投入できます。既存の旧データは削除され、新規3本で上書きされます。

```bash
# .env.local に SANITY_API_TOKEN を設定（sanity.io で発行、Editor 権限）
npm run seed:ai-demos
```

投入されるデモ: **100本**（`existingDemos` 10本 ＋ `demo-batches` 90本。うち `ai_live` 1本）。一覧は [`docs/demo-mock-inventory.md`](demo-mock-inventory.md) を参照

## 業種と基盤図の対応

| 業種 | 基盤図での表示 |
|------|----------------|
| construction | 現場案件DB + 施工判断ロジック |
| legal | 法務文書DB + 法的整合ロジック |
| manufacturing | 製造ナレッジDB + 工程最適化ロジック |

業種タグ（建設、士業、製造）を設定すると、RRINO-AI 基盤図の絞り込みに反映されます。

## 週次の無限量産フロー（追加→確認→公開）

### 運用リズム（約100本スケール）

- **推奨ペース**: 週 **3〜5件**（手早く積む場合は **10〜20件/バッチ** でまとめ投入も可）
- **バッチ単位の締め**: 各バッチ終了時に [`docs/demo-mock-inventory.md`](demo-mock-inventory.md) を更新し、[`docs/demo-mock-taxonomy-matrix.md`](demo-mock-taxonomy-matrix.md) の埋まり状況を反映する
- **品質**: 各本は [`docs/demo-mock-quality-gate.md`](demo-mock-quality-gate.md) を満たしてから Publish または `npm run seed:ai-demos`
- **データの置き場**: シードで追加する場合は [`scripts/demo-batches/`](../scripts/demo-batches/) のバッチファイルに追記し、[`scripts/seed-ai-demos.ts`](../scripts/seed-ai-demos.ts) 経由で投入（再現性・レビュー用）。Studio は個別修正用に併用可

### 基本単位

- **週3〜5件**の aiDemo を Sanity に追加
- 追加後は **即確認** してから公開

### 手順（1デモあたり）

1. Sanity Studio で AI Demo を新規作成
2. 必須項目 + 推奨項目を入力（デモ品質ゲート参照）
3. Publish
4. `/demo` 一覧に表示されるか確認
5. `/demo/[スラッグ]` で詳細・実行・CTA が正しく動作するか確認
6. 問題なければ完了

### 量産パターン（横展開）

| 機能軸 | 業種軸（例） |
|--------|--------------|
| 音声→文書化 | 建設、製造、医療・介護 |
| 画像→報告書化 | 建設、保守、不動産、保険 |
| 問い合わせ→返信 | 飲食、小売、EC、サービス |
| 検索→要約 | 士業、商社、管理部門 |

### 業種拡張が必要な場合

- 現行: construction / legal / manufacturing
- 追加例: retail / medical / logistics
- 拡張時: `src/sanity/schemaTypes/aiDemo.ts` の industry リスト、`industry-module-map.ts`、型・クエリを同期更新

## モック→実AI 昇格ルール

新規デモは原則 **mock_preview**（モック演出）で投入し、反応の良いものだけ **ai_live**（実AI）へ昇格します。

### 実行モード（runMode）

| 値 | 説明 |
|----|------|
| `mock_preview` | 事前に用意した `mockOutputPrimary` / `mockOutputSecondary` をストリーミング風に表示。AI呼び出しなし |
| `ai_live` | 実際に Gemini（Google AI）を呼び出し、リアルタイムで生成 |

### 昇格の指標例

以下のいずれかが閾値を超えたデモを `ai_live` へ昇格候補とする。

| 指標 | 目安 |
|------|------|
| **クリック率** | 一覧からの詳細遷移率が他デモの1.5倍以上 |
| **完走率** | 実行→出力完了まで到達した割合が高い |
| **問い合わせ率** | 当該デモ経由の問い合わせが発生 |

※閾値は運用で調整。まずは「反応が良い」と判断したデモから昇格。

### 昇格手順

1. Sanity Studio で該当 aiDemo を開く
2. **実行モード** を `mock_preview` → `ai_live` に変更
3. **システムプロンプト** が十分に設定されているか確認（昇格前に必須）
4. Publish
5. 実AI動作を確認（`/demo/[slug]` で実行テスト）

### 注意

- `ai_live` は API コストが発生するため、昇格は計画的に
- 昇格後も `mockOutputPrimary` / `mockOutputSecondary` は残しておくと、フォールバックやテストに利用可能

## トラブルシューティング

- **デモが一覧に表示されない**: スラッグが設定されているか確認。Publish 済みか確認
- **実行パネルが表示されない**: システムプロンプト（ai_live）またはモック出力（mock_preview）が入力されているか確認
- **API エラー**: `GOOGLE_GENERATIVE_AI_API_KEY` が設定されているか確認（ai_live 時のみ）。[Google AI Studio](https://aistudio.google.com/) でキーを取得
